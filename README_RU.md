# Android UI Flow Automator

Расширяемый фреймворк автоматизации на базе **Python** и **uiautomator2** для верификации пользовательских сценариев, мониторинга UI-метрик и стресс-тестирования навигации через Deep Links на Android.

---

## Технологический стек

| Слой | Технология |
|------|-----------|
| Язык | Python 3.12+ |
| Управление устройством | ADB (Android Debug Bridge), uiautomator2 |
| Обработка данных | Pandas, OpenPyXL |
| XPath / XML | lxml |
| Конфигурация | python-dotenv |

---

## Основные возможности

- **Интеллектуальное извлечение метрик** — парсинг числовых данных из UI с поддержкой сокращений (например, «10.5K», «19,5К» → целое число)
- **Остановка по порогу** — автоматическое завершение взаимодействия при достижении целевого значения метрики
- **Верификация Deep Links** — проверка корректного перехода из внешнего браузера во внутреннее состояние приложения
- **Ротация сессий** — циклическая многоаккаунтная авторизация для длительных тестовых прогонов
- **Адаптивная обработка UI** — перехват системных диалогов («Открыть с помощью», «Всегда», «Только сейчас») и подавление всплывающих окон
- **Управление задачами через Excel** — ведение списка задач и прогресса в локальном `.xlsx` файле

## Архитектура

```
┌─────────────────────────────────────────────────┐
│               MobileFlowEngine                  │
├─────────────────────────────────────────────────┤
│  _initialize_driver()   → Подключение по ADB    │
│  perform_auth()         → Авторизация           │
│  process_tasks()        → Основной цикл         │
│  fetch_ui_metric()      → Чтение метрик (XPath) │
│  handle_system_dialogs()→ Обработка диалогов    │
└─────────────────────────────────────────────────┘
         │                        │
    settings.py              inventory.xlsx
  (конфигурация из .env)    (таблица задач)
```

- **Разделение логики** — логика взаимодействия отделена от UI-селекторов
- **Конфигурация через окружение** — чувствительные данные хранятся в `.env`, а не в коде
- **Данные в центре** — Pandas управляет инвентарём задач и отслеживанием прогресса

## Установка и настройка

### 1. Клонирование и установка зависимостей

```bash
git clone https://github.com/OniSku/Android-UI-Flow-Automator.git
cd Android-UI-Flow-Automator
pip install -r requirements.txt
```

### 2. Конфигурация окружения

Переименуйте `.env.example` в `.env` и заполните значения:

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|
| `ADB_SERIAL` | Серийный номер устройства или адрес эмулятора | `127.0.0.1:5555` |
| `TARGET_PACKAGE` | Имя пакета целевого приложения | `com.example.app` |
| `BROWSER_PACKAGE` | Имя пакета браузера | `com.android.chrome` |
| `APP_LABEL` | Отображаемое имя приложения для системных диалогов | `TargetApp` |
| `UI_NAV_PROFILE_ID` | Resource ID элемента навигации к профилю | — |
| `UI_ACTION_BUTTON_ID` | Resource ID основной кнопки действия | — |
| `UI_LOGOUT_BUTTON_ID` | Resource ID кнопки выхода | — |
| `METRIC_KEYWORD` | Ключевое слово для поиска элемента метрики в UI | `followers` |

### 3. Подготовка файлов данных

- **inventory.xlsx** — Excel-файл со столбцами: `url`, `initial_value`, `progress`, `target`
- **credentials.txt** — по одной паре `логин:пароль` на строку

### 4. Подключение устройства

Убедитесь, что на Android-устройстве или эмуляторе включена отладка по USB/ADB:

```bash
adb devices   # проверка подключения
```

### 5. Запуск

```bash
python app_automator.py
```

Движок инициализирует драйвер, загружает инвентарь задач и начинает цикл взаимодействия.

---

## Принцип работы

1. Читает активные задачи из `inventory.xlsx` (отфильтровывает завершённые)
2. Выполняет ротацию аккаунтов из `credentials.txt`
3. Авторизуется в целевом приложении
4. Открывает URL каждой задачи в Chrome, инициируя Deep Link в целевое приложение
5. Считывает текущую UI-метрику и фиксирует начальное значение
6. Выполняет целевое действие и увеличивает счётчик прогресса
7. Удаляет задачи, достигшие порогового значения
8. Переходит к следующему аккаунту и повторяет цикл
